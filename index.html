YES Coding - Day 1: Setting Up Dev Machines

In order to set up your Development Machine from its current Windows configuration to a Linux Ubuntu config, you should complete the following steps:

  1. Partition the hardrive to make space for side-by-side Linux installation:
    1. Right click on the Windows icon (bottom left) and select 'Disk Management'.
    2. Find hard drive (usually 'C:' drive) and right click and select 'Shrink Volume'.
    3. Input partition size (in Mb) being sure not to exceed maximum available (you should not need more than 75Gb or 75,000Mb)
  2. Install Linux Ubuntu from USB zip drive:
    1. Restart Windows and hit escape before the OS begins reloading in order to bring up boot options.
    2. Select "Boot Device Options" by hitting F9 (you do not need to hit enter).
    3. Use the arrow keys to select the option "USB Hard Drive 1 - Samsung Flash Drive"
    4. Follow instructions to install Ubuntu (use Columbia Law - Guest wifi network) - Make sure that you select "Install Alongside Windows X" and do not erase existing hardrive
  3. Use Terminal to install useful development applications:
    1. Install Git: open a terminal (hit "Super Key" - the windows key - and search for "Terminal") and execute 'sudo apt-get install git'. Ask Derek for password.
    2. Install Python dev tools: Check if you have python, easy_install and pip by executing 'which python' (and so on). If the result is not '/usr/bin/python' (or /usr/bin/whatever), then install the missing applications using 'sudo apt install python-setuptools'
    3. Install Vim: Vim is a usefulll text editor, check if you have it using 'which vim'. If not, then install using 'sudo apt-get install vim'.
  4. Create Github account and pull down your first example code:
    1. Open up a browser (Firefox) and go to www.github.com
    2. Follow instructions to create a new account using an email address that you have access to.
    3. Follow instructions to create a repository and clone its contents to your local machine - this is kind of tricky, so ask questions!
    4. Make some changes to the files within your repo, add those changes to git and commit them to your 'master' branch. Push those changes back up to github and confirm the changes by viewing the repo from your web browser - again, ask many more questions!
That's it! You're on your way. Our last agenda item for today is to pick an initial project that we want to tackle.
